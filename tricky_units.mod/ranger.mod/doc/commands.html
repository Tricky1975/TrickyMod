Function ToRange[](start,eind,stap=1)
DescriptionSet up a range array for the For command.
InformationThis command has been setup as BlitzMax only accepts constants For 'step' for reasons beyond my understanding, but it is highly annoying. You don't need to set up a negative step if the end value is lower than the start value.

I know this is very likely slower than doing this the regular way, however due to BlitzMax not accepting variables for steps, I was a bit forced to come up with this!
