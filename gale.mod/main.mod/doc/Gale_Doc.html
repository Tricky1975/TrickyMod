Hi;
This is nothing more than an overview of the functions that were brought into Lua for any project you wanna start up with the GALE project.
This is by no way a guide to Lua itself as there are tons of documents about that. Just the functions that are created for Lua by GALE.

Please note, you must have GALE.bmx imported and GALE will automatically import GALE_Sys.bmx. When you import a BMX file into your BlitzMax Project, GALE will automatically add all the functions it has to Lua.
Happy coding and scripting ;)

Audio

Overview
NameAudio
TypeObject

This object contains all kinds of neat audio features.

Audio.Crash

Overview
NameAudio.Crash
TypeVariable

When set the system will crash on certain error. When set false the system will try to ignore such errors if possible.

Audio.Free

Overview
NameAudio.Free
TypeMethod
SyntaxAudio.Free(Slot)

Releases the audio buffer stored on this slot. The sound will be removed from the memory by the BlitzMax Garbage Collector eventually.

Audio.FreeChannel

Overview
NameAudio.FreeChannel
TypeMethod
SyntaxAudio.FreeChannel(Channel)

Removes a channel from the memory.

Audio.Load

Overview
NameAudio.Load
TypeMethod
SyntaxAudio.Load(Sound,Assign="")

Loads and audio file and stores it in slot 'Assign'. If the Assign value is not set the computer will assign it to a slot and return the slot name. When the load is not succesful "*ERROR*" is returned.

Audio.LoadLoop

Overview
NameAudio.LoadLoop
TypeMethod
SyntaxAudio.LoadLoop(Sound,Assign="")

Loads and audio file and stores it in slot 'Assign'. When played back the loaded sound will loop. If the Assign value is not set the computer will assign it to a slot and return the slot name.

Audio.LoadX

Overview
NameAudio.LoadX
TypeMethod
SyntaxAudio.LoadX(Sound,Loop,Hardware,Assign="")

Loads and audio file and stores it in slot 'Assign'. If the Assign value is not set the computer will assign it to a slot and return the slot name. You can use this function for some specific configuration.

Audio.Play

Overview
NameAudio.Play
TypeMethod
SyntaxAudio.Play(Slot,Channel="")

Plays a sound. If Channel is set the system will store the channel there for you.

Audio.Playing

Overview
NameAudio.Playing
TypeMethod
SyntaxAudio.Playing(Channel)

Returns 1 if a channel is playing otherwise 0. Please note that in Lua 0 counts as 'true' in boolean comparing.

Audio.QuickSound

Overview
NameAudio.QuickSound
TypeMethod
SyntaxAudio.QuickSound(Sound)

Loads an audio file from a JCR file, plays it and forgets it immediately.

Audio.Stop

Overview
NameAudio.Stop
TypeMethod
SyntaxAudio.Stop(Channel)

Stop a channel from playing. If the channel is not properly set, this routine will be ignored.

GALE_ prefix

Overview
NameGALE_ prefix
TypeBackground information

This prefix is reserved for any features that GALE may react on or assign data to. So only use GALE_ prefixed stuff in accordance with this manual. When more functions are added in the future this prefix is very likely to be put in front of it. Some projects using GALE might also use their own prefixes (for example the LAURA engine uses "__" as reserved prefix), so check the data on that.

GALE_OnLoad() function

Overview
NameGALE_OnLoad() function

GALE_OnUnload() function

Overview
NameGALE_OnUnload() function
TypeBackground information

When this function is present anywhere in the script GALE will automatically execute it when the script is removed from the memory. Please note that the BlitzMax garbage collector is responsible for this, and you never know when the garbage collector cleans up the memory as that is when the function will be called.

Image.Assign

Overview
NameImage.Assign
TypeMethod
SyntaxImage.Assign(Image,NewImageName,KeepOriginal=0)

Assigns an image to a new name so that it can be used in an easier way with other scripts.

function Main()
 img = Image.Load("MyImage.png")
 Image.Assign(img,"MyImage")
 Image.Show("MyImage",10,10)
end
Of course, the code above can also be made shorter. Like this:
function Main()
 Image.Assign(Image.Load("MyImage.png"),"MyImage")
 Image.Draw("MyImage",10,10)
end
Notes
  • You don't have to release the original image code. That happens automatically unless you set the KeepOriginal parameter.
  • If an image is already on that spot it's automatically removed. You do not have to free it first.
  • The strings you use for this feature are CASE SENSITIVE!

Image.Load

Overview
NameImage.Load
TypeMethod
SyntaxImage.Load(File)

Loads an image (from the JCR file if the project uses that. And it reacts automatically on patches) and stores it into the image buffer and storage and leaves returns an ID to find it back there.

When you load mypic.png and mypic.png.hot is loaded the picture will automatically be hotspotted according to the settings inside the .hot file. When a mypic.png.frame file exists, it'll be automatically loaded as an animated picture with the settings inside the file. When the files don't exist they will be ignored. When you want to prevent Image.Load() to check this then set the Image.DontUseSpecial variable to 1.

SELECT

Overview
NameSELECT
TypeCompiler Directive
Syntax-- @SELECT <var/function>

You can use this directive to create a "SELECT" statement (same as "switch" in C-syntax languages). As Lua doesn't support that itself I made this preprocessor do it.
You cannot place a remark behind this statement or the preprocessor will create faulty Lua scripts. If you want a remark with this, put it either above or below it :)
NOTE: Only one SELECT can be open at a time. When you open a new one, the old one will be closed, however keep closing your SELECT statements anyway, as you never know if I might improve this in the future :)

Opens a dialog box that asks the user for confirmation.

Var.S

Overview
NameVar.S
TypeMethod
SyntaxVar.S(St)

Will automatically fill out all vars in a given string and return that.

Var.D("$NAME","Jeroen")
a = Var.S("My name is $NAME") -- Lua variable a will now contain 'My name is Jeroen'
IMPORTANT NOTE: It must be noted that the system sees no diffrence between text and variables. As soon as it recognizes a variable it will start replacing. That's why I prefer to start all my vars with $ (like in php). Sometimes I another char that put itself apart. This way you can prevent a lot of misery. Oh yeah, and the variable names are CASE SENISTIVE!!
