GALE.MGUI: Globals Modinfo Source  

This file contains a driver to make GALE operate from a GUI environment.

Please note that not all of Gale's sub modules will operate properly from this environment.

NEVER import this module together with GALE.M2D or your project will cause conflicts which will lead to very terrible results in the worst case even system crashes.

Globals Summary

GALE_ConsoleGadget , GALE_ExitGadget , GALEMGUISTDOUT

Globals

Global GALE_ConsoleGadget:TGadget
DescriptionWhen set to 'null' (default value) all console output requests will be ignored. When a textfield gadget is tied to this variable, a log will be stored in there for the user of your program to see (if that gadget is visible at the time that is).

Global GALE_ExitGadget:TGadget
DescriptionThis variable should be set to the Window gadget your ending console is part of. Then the system will show this window when a Lua error pops up and when this window is closed the app will end. When not set nothing will be shown in case of a Lua error. (When running your BlitzMax program from the console you should see the error on the console, if you turned the GALEMGUISTDOUT variable to 'true' that is.

Global GALEMGUISTDOUT
DescriptionIf set to true all console output will be displayed on the STD OUT output as well. Handy when you work from the MaxIDE or similar programs ;)
